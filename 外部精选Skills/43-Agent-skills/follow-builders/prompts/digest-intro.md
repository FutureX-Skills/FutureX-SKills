# 日报格式规范

你正在把各个信息源的摘要组装成一份完整的日报。

## 格式模板

```
# AI 构建者日报 — [日期]

---

## X / Twitter

### [人名] — [角色/公司]
[摘要内容，2-4 句话]
[链接]

### [人名] — [角色/公司]
...

---

## 播客

### [播客名] —「[集标题]」
**一句话要点**：[最核心的收获]

[摘要正文]

[链接]

---

> 回复可随时调整投递设置、关注列表或摘要风格。
```

## 格式规则

- 用 Markdown 标题层级组织（`##` 大分类，`###` 每个人/每期）
- 每个信息源之间空一行，保持呼吸感
- 播客摘要开头必须有加粗的「一句话要点」
- 手机阅读友好：段落短、层次清晰、不堆砌

## 内容规则

### 推文作者格式
- 用全名 + 角色/公司（如「Replit CEO Amjad Masad」，不要只写姓）
- 不要写 @handle（Telegram 上会变成错误的链接）。写「Aaron Levie (levie on X)」或直接用全名
- 每条推文必须附 `url` 字段的原始链接

### 播客链接
- 用 JSON `url` 字段的具体视频链接（如 `https://youtube.com/watch?v=xxx`）
- 不要链接到频道主页
- 标题用 JSON `title` 字段原文

### 链接是底线
- 每条内容必须有原始链接。没有链接 = 不包含
- 推文：`https://x.com/handle/status/xxx`
- 播客：YouTube 视频链接

### 绝不编造
- 只用 feed JSON 中的真实内容
- 不要编造引言、观点或猜测某人的想法
- 不要对某人的「沉默」做解读
- 没有真实内容的构建者直接跳过
