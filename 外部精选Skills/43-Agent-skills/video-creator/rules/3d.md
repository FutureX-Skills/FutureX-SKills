# Using Three.js and React Three Fiber in Remotion

Follow React Three Fiber and Three.js best practices.  
Only the following Remotion-specific rules need to be followed:

## Prerequisites

First, the `@remotion/three` package needs to be installed.  
If it is not, use the following command:

```bash
npx remotion add @remotion/three # If project uses npm
bunx remotion add @remotion/three # If project uses bun
yarn remotion add @remotion/three # If project uses yarn
pnpm exec remotion add @remotion/three # If project uses pnpm
```

## 渲染必须加 `--gl=angle`

3D 内容在 headless Chrome 中渲染，**必须**指定 GPU 后端：

```bash
npx remotion render --gl=angle src/index.ts MyComp out/video.mp4
npx remotion still --gl=angle src/index.ts MyComp test.png
```

不加 `--gl=angle` 会导致 WebGL context 创建失败，输出白屏或黑屏。  
Studio 预览不需要此参数。

## Using ThreeCanvas

You MUST wrap 3D content in `<ThreeCanvas>` and include proper lighting.  
`<ThreeCanvas>` MUST have a `width` and `height` prop.

```tsx
import { ThreeCanvas } from "@remotion/three";
import { useVideoConfig } from "remotion";

const { width, height } = useVideoConfig();

<ThreeCanvas width={width} height={height}>
  <ambientLight intensity={0.4} />
  <directionalLight position={[5, 5, 5]} intensity={0.8} />
  <mesh>
    <sphereGeometry args={[1, 32, 32]} />
    <meshStandardMaterial color="red" />
  </mesh>
</ThreeCanvas>;
```

## No animations not driven by `useCurrentFrame()`

Shaders, models etc MUST NOT animate by themselves.  
No animations are allowed unless they are driven by `useCurrentFrame()`.  
Otherwise, it will cause flickering during rendering.

Using `useFrame()` from `@react-three/fiber` is forbidden.

## Animate using `useCurrentFrame()`

Use `useCurrentFrame()` to perform animations.

```tsx
const frame = useCurrentFrame();
const rotationY = frame * 0.02;

<mesh rotation={[0, rotationY, 0]}>
  <boxGeometry args={[2, 2, 2]} />
  <meshStandardMaterial color="#4a9eff" />
</mesh>;
```

## Using `<Sequence>` inside `<ThreeCanvas>`

The `layout` prop of any `<Sequence>` inside a `<ThreeCanvas>` must be set to `none`.

```tsx
import { Sequence } from "remotion";
import { ThreeCanvas } from "@remotion/three";

const { width, height } = useVideoConfig();

<ThreeCanvas width={width} height={height}>
  <Sequence layout="none">
    <mesh>
      <boxGeometry args={[2, 2, 2]} />
      <meshStandardMaterial color="#4a9eff" />
    </mesh>
  </Sequence>
</ThreeCanvas>;
```

## 纹理加载：必须在 ThreeCanvas 外部

**关键规则**：`useDelayRender()` 在 ThreeCanvas 内部不生效，因为 ThreeCanvas 有独立的 React Fiber 上下文。纹理必须在 ThreeCanvas **外部**加载，通过 props 传入内部组件。

### 错误做法（会导致渲染时纹理丢失）

```tsx
// 不要这样做！ThreeCanvas 内部的 useDelayRender 不工作
const MyScene = () => {
  const texture = useLoader(THREE.TextureLoader, "/texture.jpg"); // 不可靠
  return <mesh><meshStandardMaterial map={texture} /></mesh>;
};

<ThreeCanvas><MyScene /></ThreeCanvas>
```

### 正确做法

```tsx
import { useState, useEffect } from "react";
import { useDelayRender } from "remotion";
import * as THREE from "three";

// 步骤 1：在 ThreeCanvas 外部加载纹理
const useTexture = (url: string) => {
  const [texture, setTexture] = useState<THREE.Texture | null>(null);
  const { delayRender, continueRender } = useDelayRender();
  const [handle] = useState(() => delayRender());

  useEffect(() => {
    const img = new Image();
    img.crossOrigin = "anonymous";
    img.onload = () => {
      const tex = new THREE.Texture(img);
      tex.needsUpdate = true;
      setTexture(tex);
      continueRender(handle);
    };
    img.src = url;
  }, [url]);

  return texture;
};

// 步骤 2：ThreeCanvas 内部组件通过 props 接收纹理
const Scene: React.FC<{ texture: THREE.Texture | null }> = ({ texture }) => {
  const frame = useCurrentFrame();
  return (
    <mesh rotation={[0, frame * 0.01, 0]}>
      <sphereGeometry args={[2, 64, 64]} />
      <meshStandardMaterial map={texture} />
    </mesh>
  );
};

// 步骤 3：组合
export const My3DScene: React.FC = () => {
  const texture = useTexture(staticFile("texture.jpg"));
  const { width, height } = useVideoConfig();

  return (
    <ThreeCanvas width={width} height={height}>
      <ambientLight intensity={0.5} />
      <directionalLight position={[3, 2, 8]} intensity={1.5} />
      <Scene texture={texture} />
    </ThreeCanvas>
  );
};
```

**核心原则**：所有需要 `useDelayRender` 的异步操作（纹理、模型、数据）都必须在 Remotion 的 React 树中完成（ThreeCanvas 外部），然后通过 props 传入 Three.js 场景。

## 光源位置

默认摄像机在 z 轴正方向。确保主光源在物体**面向摄像机**的一侧，否则物体会显得全黑：

```tsx
// 好：光源在摄像机前方，照亮物体正面
<directionalLight position={[3, 2, 8]} intensity={1.5} />

// 差：光源在物体背后，正面全黑
<directionalLight position={[0, 0, -5]} intensity={1.5} />
```

建议总是搭配环境光，避免未照到的面全黑：

```tsx
<ambientLight intensity={0.4} />
<directionalLight position={[3, 2, 8]} intensity={1.5} />
```
