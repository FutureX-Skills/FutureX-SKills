# AI 热点深度长文自动生成技能

本技能用于自动采集前一日 AI 行业热点，智能选题，并输出**不少于5000字**的中文深度长文，配有图表，最终以排版精美的 Word 文档交付。

---

## 字数硬性要求

> **每次输出正文字数必须 >= 5000字（不含标题、图注、来源等辅助内容）**

字数分配方案：
- 主选题深度文章：3000-4000字
- 次选题扩展报道（2-3篇）：每篇 600-800字（合计 1200-2400字）
- 行业观察板块：500-700字
- 读者行动指南：300-500字

**字数自检规则**：完成内容写作后，逐项核查各板块字数。任一项未达标，必须继续补充，禁止以"内容已充分"为由提前截止。

---

## 整体工作流程

```
Step 1: 热点采集（至少10次搜索 + web_fetch深度抓取）
    ↓
Step 2: 智能选题（评分筛选，1主题 + 2-3副题）
    ↓
Step 3: 深度内容创作（严格按框架展开，总字数 >= 5000字）
    ↓
Step 4: 图表生成（至少2张，Python matplotlib）
    ↓
Step 5: Word 文档输出（docx-js，必须阅读 docx SKILL.md）
```

## Step 1：热点采集

使用 `web_search` + `web_fetch` 工具搜索昨日AI领域重要事件。

### 必搜关键词组合（至少执行以下8组）

```
1. "AI 人工智能 新闻 {昨天日期}"
2. "大模型 发布 更新 {本周}"
3. "OpenAI Anthropic Google AI Meta news {date}"
4. "AI Agent 智能体 最新进展"
5. "人工智能 国内 进展 {昨天日期}"
6. "AI 融资 投资 {月份}"
7. "AIGC 生成式AI 应用 {本周}"
8. "AI 监管 政策 {月份}"
9. "AI 芯片 算力 {近期}"
10. "机器学习 技术突破 {近期}"
```

### 抓取规则

- 每条候选新闻用 `web_fetch` 获取完整正文（不能只看标题摘要）
- 优先来源：机器之心、量子位、36氪AI频道、InfoQ、TechCrunch、雷峰网
- 收集 15-25 条候选新闻，每条记录：标题、来源、时间、核心摘要（100字）、数据亮点、热度信号、可延伸的深度角度

## Step 2：智能选题

### 评分维度（满分100分）

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 新鲜度 | 25% | 24h内=25分；48h内=15分；更早=5分以下 |
| AI相关性 | 20% | 核心属于AI/大模型/AIGC=20分；相关行业=10分 |
| 用户价值 | 25% | 对国内从业者有直接参考价值=25分；泛知识=10分 |
| 热度信号 | 15% | 有明确社媒热议证据=15分；无证据=0-5分 |
| 深度潜力 | 15% | 可展开3000字以上深度分析=15分；只能写500字=0分 |

### 选题策略

- **主选题**（1个）：综合评分最高，必须能展开3000字以上
- **次选题**（2-3个）：每个能扩展 600-800字
- 选题后先规划写作提纲（至少5个二级标题），确认字数可行再开始写

### 排除规则

- 排除：纯英文学术论文（除非引发国内广泛讨论）
- 排除：无实质内容的PR软文
- 排除：政治高度敏感话题
- 排除：无法展开到600字以上的简讯

## Step 3：深度内容创作（核心步骤）

### 3.1 主文章框架（目标：3000-4000字）

必须包含以下全部章节，每节最低字数如标注：

```
【文章标题】（20字以内，含核心关键词，有冲击力）
【导语】（100-150字：背景 + 核心结论 + 阅读价值）

第一章：事件全景（400-600字）
  1.1 发生了什么（时间线，引用具体数据）
  1.2 关键数字/参数（量化呈现，不用模糊描述）
  1.3 各方反应（行业人士/媒体/用户的初步反馈）

第二章：深度技术/商业解读（800-1000字）
  2.1 技术原理浅析（用类比帮助非技术读者理解）
  2.2 与现有方案的横向对比（同类产品/技术）
  2.3 真正的突破点（找到1-2个核心创新）
  2.4 局限性与挑战（不回避缺点，体现客观性）

第三章：行业影响与竞争格局（600-800字）
  3.1 对国内行业的直接影响
  3.2 竞争对手如何应对
  3.3 产业链上下游影响
  3.4 资本市场信号

第四章：国内外对比与启示（500-700字）
  4.1 国内同类进展（类比国内案例/差距）
  4.2 对国内企业/从业者的启示
  4.3 政策与监管视角

第五章：展望与预测（400-500字）
  5.1 短期（3-6个月）可能发生什么
  5.2 长期趋势判断
  5.3 值得持续关注的信号
```

### 3.2 次选题扩展报道（每篇 600-800字）

每个次选题按以下精简框架展开：

```
【小标题】（10字以内）
【一句话导语】

核心事件（200字）：发生了什么，关键数据
为什么重要（200字）：对行业/用户的意义
深度延伸（200-300字）：技术解读 OR 竞争分析 OR 场景预测
一句话结论（50字以内）
```

### 3.3 行业观察板块（500-700字）

在所有选题文章之后，必须增加"本周AI行业观察"板块：

```
趋势信号（200-250字）：从所有热点中提炼2-3个宏观趋势

值得关注的冷门进展（150-200字）：1-2条未被主流媒体大量报道但有潜在价值的新闻

数据速览（100-150字）：本周AI领域重要数字：融资金额、用户量、Benchmark分数等
```

### 3.4 读者行动指南（300-500字）

文章末尾必须包含针对不同读者群体的行动建议：

```
对于产品经理/创业者：（80-100字）
对于开发者/工程师：（80-100字）
对于投资者/行业观察者：（80-100字）
对于普通AI爱好者：（50-80字）
```

### 3.5 写作质量标准

| 维度 | 要求 |
|-----|------|
| 数据驱动 | 每个核心论点至少1个量化数据支撑 |
| 类比通俗 | 每个技术概念至少1个生活化类比 |
| 观点明确 | 不能只罗列事实，必须有判断和立场 |
| 国内视角 | 每章至少1处结合国内场景或案例 |
| 可读性 | 段落不超过150字，避免大段连续文字 |
| 英文缩写 | 首次出现括号注明中文全称 |

## 常见问题

**Q: 单个选题信息量不足写3000字怎么办？**
A: 按框架强制展开：第二章增加"技术背景深挖"（额外搜索技术文档）、"历史横向对比"（类似事件回顾）、"用户视角场景分析"三个子节。

**Q: 找不到24h内足够热点怎么办？**
A: 扩展到72h，行业观察板块扩充至800-1000字，确保总字数达标。

**Q: 图表字体乱码怎么办？**
A: 图表标签全改英文，在Word图注中用中文详细说明。

**Q: 如何避免字数多但内容空洞？**
A: 每增加100字，检验：这段为读者新增了什么信息或观点？若无法回答则删除重写。字数靠深度，不靠重复。
